# 游戏基础系统模块说明文档

包含：异步日志系统、埋点追踪系统、音频播放与音量控制组件

日志文件的路径在 build/Release/log/log.txt

也就是.exe目录下的log文件夹里面

## 一、 异步日志系统 

### 功能概述

日志系统采用  单例模式 + 异步消费者线程 (QThread) 的架构。应用各处可以通过宏 LOG_INFO, LOG_WARN, LOG_ERR 将日志以极低的延迟压入队列(QQueue)，由后台专属线程负责真实的文件磁盘写入，保证了在繁重的游戏帧渲染过程中，日志写入不会造成主线程的卡顿。

### 遇到的问题与解决方案

#### 问题一：日志文件“消失”或存储路径不受控

现象： 最初使用 QDir::currentPath() 获取当前工作目录，导致依赖 IDE 启动或不同构建系统（如 Ninja构建目录）时，日志文件 app_log.txt 被胡乱生成在 out/build/debug/.. 等隐藏深层的临时目录中。

解决： 改用 QCoreApplication::applicationDirPath() 获取绝对的执行文件路径，并在其旁边强制判断并使用 QDir::mkdir("log") 新建并锁定 log/log.txt。确保了日志文件始终与执行文件同级，稳定可靠。

#### 问题二：包含中文的日志写入会导致整个日志系统“永久罢工”

现象： 在写入纯英文（如“Game starting”）时一切正常，但在写入拼接着中文（如“第 1 关 使用按键...正确率...”）的日志后，后续哪怕是纯英文也会写入失败，日志看似被阶段性截断。
解决： 经查，由于 Windows 系统的默认编码问题，QTextStream 在处理 QString 到本地文本流时抛出了解析异常，导致流状态崩溃且无法恢复。我们将 QTextStream 彻底抛弃，直接调用底层的 file.write((msg + "\n").toUtf8())，将中文字符串强制转为 UTF-8 字节流后进行二进制注入，完美跨越了编码障碍。

#### 问题三：程序被强制关闭导致尾部日志丢失

现象： 当应用被直接关掉时，残留在系统 IO 缓存中的日志来不及真实写入硬盘。
解决： 在线程写入循环中增加 file.flush();，强迫操作系统在每次弹出队列写入后立即进行硬盘缓存刷新，防止强杀丢日志。

## 二、 数据埋点与统计系统

### 1.功能概述

Tracker 同样依托后台异步线程，主要用于记录玩家的行为事件（Event & Data），如： Game_Start，Game_Over，Apple_Hit。配合游戏控制器 (GameController)，它还会深度统计玩家当前关卡的表现：按了哪些键、正确率是多少、失败了多少次，为游戏数值平衡和玩家行为分析提供数据依据。

### 2.遇到的问题与解决方案

#### 问题一：玩家中途退出或重开导致半途数据丢失

现象： 埋点结算原本只放置在玩家过关升难度 或者 生命扣光彻底 Game Over 这两种自然结算条件下。如果玩家打到一半点击了界面上的[退出]、[结束]、[重新开始]，或者直接点了窗口的 [X] 强转关闭，这部分包含大量按键的练习数据就会彻底丢失。
解决： 在 GameController 中对 UI 信号 (sigExit, sigEnd, sigStart) 的回调进行了拦截，在此处加入了带条件触发的 logLevelStats(true)。同时，为其加入了 ~GameController() 析构函数兜底。只要系统被销毁，且本生命周期内有按过键盘（m_totalPresses > 0），就会强制计算正确率并进行拦截上报。

#### 问题二：流写入崩溃问题

解决： Tracker 同样存在与 AsyncLogger 相同的中文写入瘫痪问题，使用修改原生 Qt.toUtf8() 的同款修复方案。

## 三、 音频与音量控制系统 

#### 功能概述

基于 QMediaPlayer 封装封装的全局单例控制器。

内部维护了两条独立音轨： bgPlayer (BGM背景音乐) 和 fxPlayer (音效/吃苹果声)。

提供了暴露给外部的快捷 playSound(SoundType) 接口，以及由设置菜单滑块直接映射调整的 setVolume(int) 接口。

#### 遇到的问题与解决方案

问题一：设置菜单修改音量无法全局同步生效

现象： 各个视图自行创建播放器或独立调用，设置菜单拉动后难以通知底层的媒体组件。
解决： 彻底收拢权限到 AudioSystem 单例中。在设置界面 QSlider (音量滑动条) 槽函数中，使用 Lambda 表达式直接绑定 AudioSystem::instance().setVolume(value)，拉动滑块瞬间，BGM 和所有触发音效齐步调节。
