## **1.在函数后面加上const 表示这个函数只读取，不会改变变量**

## **2.size_t 与 int的区别**  

(A) 语义正确：它表示“数量/容量”

`m_freeCapacity` 表示“还剩多少块可分配”，这类值在 C/C++ 里惯例用 **无符号的大小类型**：

- `size_t` 就是专门用来表示**大小、元素个数、数组下标**的类型
- 标准库里几乎所有“长度/大小”相关接口（如 `vector::size()`、`sizeof`）都用 `size_t`

(B) 取值范围更大

在 64 位程序里：
- `size_t` 是 8 字节（0 ~ 2^64-1）
- `int` 是 4 字节（约 -21亿 ~ 21亿）

- **`size_t`**：用来表示“大小/数量/下标”，范围大，和标准库一致。
- **成员函数加 `const`**：表示“只读函数”，不会修改对象；也让 `const` 对象/`const&` 参数可以调用它。

## 3.关于构造函数拷贝问题

- `Chunk* m_pool;` 指向 `new Chunk[...]` 得到的数组
- `Chunk* m_freeList;` 指向池内部链表

如果不禁止拷贝构造和拷贝赋值，编译器会自动生成“默认拷贝”（**浅拷贝**），等价于把这些指针值直接复制：

- 拷贝后：两个 `MemoryPool` 对象都指向同一块 `m_pool` 内存
- 两个析构函数都会执行 `delete[] m_pool;`
- 结果就是：**同一块内存被 delete[] 两次 → 未定义行为导致崩溃）**

举例（不禁止拷贝时会出事）：

C++

```
MemoryPool<Bullet> p1(100);
MemoryPool<Bullet> p2 = p1;   // 浅拷贝：p2.m_pool == p1.m_pool
// 离开作用域时 p1、p2 都 delete[] 同一地址 → 崩
```

所以：**管理资源的类默认都不应该允许拷贝**，除非你自己实现“深拷贝”（为新对象重新分配内存并复制状态），但对内存池来说深拷贝通常没意义且很复杂。
