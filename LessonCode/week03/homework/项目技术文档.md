# 游戏简介

## 1. 项目概述

本项目是一个基于 **Qt Widgets** 的桌面小游戏集合，目前已实现：

- **主菜单 MainMenu**
  - “Save the Apple”模式（已实现）
- **Save the Apple（AppleGame）**
  - 随机生成带字母的“苹果”从屏幕顶部下落
  - 用户按键（A-Z）击中对应字母，消除苹果并累计成功次数
  - 苹果下落到指定失败线（窗口高度 70%）则判定失败并计数
  - 支持暂停/开始（按钮控制）
  - 提供关卡、分数、命中率等 HUD 统计显示
  - 达到目标成功数后自动进入下一关（提升难度）

---

## 2. 技术栈与依赖

- 语言：C++
- UI 框架：Qt Widgets
- 关键 Qt 类：
  - `QApplication`：应用入口与事件循环
  - `QWidget`：窗口基类
  - `QPushButton`：按钮控件
  - `QTimer`：定时器驱动游戏循环（约 60 FPS）
  - `QPainter`：2D 绘制（HUD + 苹果绘制）
  - `QRandomGenerator`：随机生成位置与字母
  - `QMessageBox`：提示框（Space Wars 占位）

## 4. 启动流程与窗口切换

### 4.1 程序入口（main.cpp）

- 创建 `QApplication`
- 创建 `MainMenu` 并 `show()`
- 进入事件循环 `a.exec()`

### 4.2 主菜单（MainMenu）

主菜单在构造时完成：

1. `uiInit()` 初始化 UI：
   - 设置窗口大小 `800x600`
   - 设置背景为资源图片 `APPLE_BACKGROUND.png`
   - 两个按钮：`btnApple`、`btnSpace`

2. 信号槽连接：
   - `btnApple.clicked -> startGameApple()`
   - `btnSpace.clicked -> startGameSpace()`

#### 4.2.1 进入 AppleGame

`startGameApple()` 中逻辑：

- `AppleGame *game = new AppleGame;`
- `game->setAttribute(Qt::WA_DeleteOnClose);`  
  关闭窗口时自动释放对象（避免内存泄漏）
- `game->show();`
- `this->close();` 关闭主菜单窗口

## 5. AppleGame 游戏机制说明

### 5.1 核心概念与状态机

AppleGame 通过一个状态变量 `state` 控制行为，至少包含：

- `Playing`：运行中（更新逻辑生效、按键生效）
- `Paused`：暂停（更新逻辑停止、按键忽略）
- `End`：结束（当前代码中用于“过关瞬间”切换）

在 `updateGame()` / `keyPressEvent()` 中都显式判断：
- `if (state != Playing) return;`

这保证了暂停或结束状态下不会继续更新和接受打字击中。

### 5.2 UI 与输入焦点

- `setFixedSize(800, 600)` 固定窗口大小
- `setFocusPolicy(Qt::StrongFocus)` 允许窗口接收键盘事件
- `onbtnStartClick()` 中调用 `this->setFocus()`，用于从暂停恢复后重新获得焦点，确保键盘输入可用

### 5.3 游戏循环（QTimer 驱动）

在构造函数中：

- 创建 `QTimer`
- `timeout -> updateGame()`
- `timer->start(16)` 约 60 FPS

该模型属于典型的“定时 tick”驱动，逻辑与绘制分离：
- tick 更新游戏对象（位置、碰撞/判定、生成）
- `update()` 触发 Qt 重绘，最终调用 `paintEvent()`

### 5.4 苹果生成 spawnApple()

生成规则：

- 若 `apples.size() >= m_maxApples` 则不再生成（控制屏幕最大对象数）
- 初始位置：
  - `x`：随机 `[100, width()-100]`
  - `y`：固定 `-50`（从屏幕外上方出现）
- 字母唯一性：
  - 在当前 `apples` 列表中保证 `letter` 不重复
  - 随机 A-Z，直到找到未被占用的字母

### 5.5 更新逻辑 updateGame()

仅在 `Playing` 状态下执行：

1. 速度与关卡关联  
   - `speed = 1.0 + level * 0.5`

2. 随机生成概率  
   - 每帧 `2%` 概率尝试生成一个苹果（同时受 `m_maxApples` 限制）

3. 移动与失败判定  
   - 失败线：`failY = height() * 0.7`
   - 苹果 y 增加 `speed`
   - 若 `y >= failY`：
     - `m_failCount++`
     - 从列表移除该苹果

4. 刷新绘制  
   - `update()` 触发重绘

### 5.6 键盘命中 keyPressEvent()

仅在 `Playing` 状态下处理：

- 只响应 `A-Z`
- 每次有效按键：`m_totalPresses++`
- 查找列表中是否存在对应字母苹果：
  - 找到则：
    - `m_successCount++`
    - 移除该苹果

#### 5.6.1 过关与难度提升

当 `m_successCount >= m_targetSuccess`：

- `m_level++`
- `m_maxApples++`（屏幕允许更多苹果）
- `m_targetSuccess += 10`（下一关目标更高）
- `m_successCount = 0`
- `apples.clear()`
- 状态回到 `Playing`

当前实现属于“自动下一关”，无弹窗确认（注释中保留了 `QMessageBox`）。

### 5.7 绘制 paintEvent()

绘制内容包括：

1. 失败线（红色）  
   - y = `0.7 * height()`

2. 苹果绘制  
   - 使用图片 `APPLE_NORMAL.png` 绘制到 `QRectF(a.x, a.y, 40, 40)`
   - 文字（字母）绘制在苹果矩形中心

3. HUD 统计（左上角）
   - Score：`success*10 - fail*5`
   - Accuracy：`success / totalPresses * 100%`
   - 显示格式：
     - `Level | Score | Success current/target | Fail | Accuracy`

## 6. 交互按钮说明（AppleGame）

AppleGame 创建了一组按钮并绑定槽函数：

- Exit：`onbtnExitClick()` -> `close()`
- Stop：`onbtnStopClick()` -> `state = Paused`
- Start：`onbtnStartClick()` -> `state = Playing` + `setFocus()`
- End：`onbtnEndClick()`（当前未实现）
- Set：`onbtnSetClick()`（当前未实现）

## 7. 关键数据与可配置项

在 `AppleGame::setApple(...)` 中可设置：

- `m_level`：关卡（>0 才生效）
- `m_successCount`：当前成功数
- `m_failCount`：失败数
- `m_totalPresses`：总按键数（用于命中率）
- `m_targetSuccess`：目标成功数
- `m_maxApples`：屏幕最大苹果数量

为后续“设置面板/存档/继续游戏”提供了接口基础。
